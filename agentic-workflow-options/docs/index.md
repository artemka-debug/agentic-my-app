# Agentic Workflow Options

This directory organizes the local-first Cursor-centered workflow research and implementation plans produced by the planning subagents.

## Options

- [Cursor Native Local Workflow](../cursor-native/docs/plan.md) - Cursor Skills, Rules, Hooks, Worktrees, /best-of-n, local verification, and GitHub PR output.
- [Cursor SDK Local Orchestrator](../cursor-sdk-local-orchestrator/docs/plan.md) - Custom TypeScript orchestrator using the Cursor SDK local runtime only.
- [LangGraph Local Orchestrator](../langgraph-local-orchestrator/docs/plan.md) - LangGraph state-machine orchestration with Cursor as the local harness.
- [Aider Workers](../aider-workers/docs/plan.md) - Cursor harness with Aider as local implementation workers in isolated worktrees.
- [OpenHands Local Runtime](../openhands-local-runtime/docs/plan.md) - Cursor harness using OpenHands local or local Docker runtimes as implementation engines.
- [mini-SWE-agent / SWE-agent Workers](../mini-swe-agent-workers/docs/plan.md) - SWE-style local issue-solving workers coordinated by Cursor.
- [Local CLI Workers: OpenCode, Claude Code, Amp](../local-cli-workers-opencode-claude-amp/docs/plan.md) - Replaceable local CLI worker engines launched by a Cursor-controlled harness.
- [IDE Agent Inspirations: Roo, Cline, Continue, Qodo](../ide-agent-inspirations-roo-cline-continue-qodo/docs/plan.md) - Adapting ideas from IDE-local agents while keeping Cursor as the harness.
- [Method Frameworks: BMAD and Agent OS](../method-frameworks-bmad-agent-os/docs/plan.md) - Using BMAD as process layer and Agent OS as the spec/context substrate.

## Pending Or Unavailable Specs

None. All requested planning subagent specs were available and written to their option folders.
